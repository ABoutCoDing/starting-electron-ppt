Electron의 핵심 이해하기

main process / renderer process

Multi-process Architecture

https://www.chromium.org/developers/design-documents/multi-process-architecture
    main process
    • 전체적으로 모든 것들을 관리하는 프로세스 (눈에 안 보임!)
    • 시스템에 접근할 수 있는 프로세스
    renderer process
    • 실제 화면에 보여지는 부분
    • Chrome 에서의 각각의 '탭'에 해당
    • 시스템에 접근할 수도 있는 프로세스 (권장 X)

    Angular2 : TypeScript 기반.
    컴파일 과정이 너무 복잡해요.
    TypeScript -> (NG Compile)TypeScript -> (Transpile)ES6 -> (Treeshaking)ES6 -> (Transpile)ES5
    이 과정에서 브라우저는 (아직까진) ES6 모듈을 지원하지 않기 때문에, import / export 구문들은 모두 ES5로 트랜스파일 되어야 해요!
    하지만 Electron 의 모듈에 대한 컨텍스트와, Angular 의 모듈에 대한 컨텍스트가 서로 다르기 때문에 이를 맞춰주려면 많은 설정을 해야 해요.
    간단하게는 webpack 의 target: 'electron-render' (1.14 이상부터!) 복잡하게는 연관된 모든 설정들!!
